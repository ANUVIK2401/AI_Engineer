01Beginner

Supervised vs Unsupervised vs Reinforcement Learning

These three paradigms define how a model learns — not what architecture it uses. Picking the wrong paradigm before you start is the most expensive mistake in ML; no amount of tuning will fix a fundamentally mismatched approach.

ParadigmHow it learnsData neededClassic algorithmsWhen to use
Supervised From labelled input→output pairs Labelled dataset — expensive to get Linear/logistic regression, trees, neural nets Spam detection, image classification, price prediction
Unsupervised Finds structure in unlabelled data Unlabelled data — cheap and plentiful k-Means, PCA, autoencoders, DBSCAN Customer segmentation, anomaly detection, dim reduction
Reinforcement Agent takes actions, gets reward/penalty An environment to interact with Q-learning, PPO, DQN, AlphaZero Games, robotics, recommendation ranking, ad bidding
Self-supervised Creates its own labels (predict next token) Massive unlabelled corpus GPT, BERT, CLIP, LLaMA Pre-training LLMs, vision-language models
🏢 Real-world: Netflix uses supervised learning for churn prediction, unsupervised for content clustering, and RL to optimise which thumbnail to show each user.
💡
Key Insight Self-supervised learning is why LLMs exist. By predicting the next word on trillions of tokens, models learn rich representations without a single human label — it's the bridge between cheap unlabelled data and powerful supervised behaviour.

Semi-supervised

Small labelled set + large unlabelled set. Label propagation, pseudo-labelling. Common when labelling is expensive (medical images).

Transfer Learning

Pre-train on large dataset, fine-tune on small target. Used for 95%+ of real-world DL tasks. BERT → fine-tune for sentiment.

RLHF

Reinforcement Learning from Human Feedback. How ChatGPT was fine-tuned to be helpful. Combines supervised (SFT) + RL (PPO).

Few-shot Learning

Learn from 1–5 examples per class. Meta-learning trains the model to learn fast. Prototypical networks, MAML.

02Beginner

Linear Regression

The simplest and most interpretable ML model — fit a straight line through data to predict a continuous value. It's the baseline every more complex model must beat, and it directly teaches you loss functions, gradients, and overfitting in the clearest possible setting.

$\hat{y} = w_1 x_1 + w_2 x_2 + \cdots + w_n x_n + b \;\;=\;\; \mathbf{w}^T \mathbf{x} + b$
Interactive — drag sliders to refit the line (red dashes = residuals, MSE updates live)
w = 0.70
b = 0.00
python 
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error

model = LinearRegression()
model.fit(X_train, y_train)

print(model.coef_)      # weights  → shape (n_features,)
print(model.intercept_) # bias b

preds = model.predict(X_test)
mse   = mean_squared_error(y_test, preds)
r2    = model.score(X_test, y_test)  # 1.0 = perfect fit

✓ Use when

  • Target is a continuous number (price, score, temperature)
  • You want an interpretable, fast baseline
  • Relationship is roughly linear
  • Coefficients need to be explainable to stakeholders

✗ Don't use when

  • Target is a category (use logistic regression or trees)
  • Relationship is highly non-linear
  • Severe multicollinearity in features (use Ridge/Lasso)
  • Outliers are extreme and uncleaned
🏢 Real-world: Zillow's Zestimate started as linear regression on square footage, location, and age. Even complex pipelines today still include linear components for interpretable price adjustments.
💡
Key Insight MSE loss + linear model has a closed-form solution: $\mathbf{w} = (\mathbf{X}^T\mathbf{X})^{-1}\mathbf{X}^T\mathbf{y}$ — no gradient descent needed. But when features exceed ~10,000 or the matrix is near-singular, you must use gradient descent. That's why sklearn switches solvers automatically for large problems.
03Beginner

Logistic Regression

Despite the name, logistic regression is a classification algorithm. It squashes a linear score through the sigmoid function to output a probability between 0 and 1. It's also the foundation of every neural network output layer for classification tasks.

$\sigma(z) = \dfrac{1}{1 + e^{-z}} \qquad\text{where}\qquad z = \mathbf{w}^T\mathbf{x} + b$
Sigmoid curve — maps any real number to a probability (0–1)
z = 0.0 → p = 0.500
python 
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import classification_report

model = LogisticRegression(max_iter=1000, C=1.0)
# C = 1/λ — smaller C = stronger L2 regularisation
model.fit(X_train, y_train)

# Get calibrated probabilities, not just hard 0/1 labels
probs  = model.predict_proba(X_test)[:, 1]
labels = (probs > 0.5).astype(int)   # tune threshold per use case

print(classification_report(y_test, labels))

✓ Use when

  • Binary or multi-class classification
  • You need calibrated probabilities, not just labels
  • Fast, interpretable baseline required
  • Features are linearly separable or close to it

✗ Don't use when

  • Classes are highly non-linearly separable
  • Very high-dimensional sparse text (try linear SVM)
  • Complex feature interactions dominate (use trees)
🏢 Real-world: Gmail's spam filter at its core is logistic regression — fast to retrain daily on new patterns, and outputs a calibrated probability that maps cleanly to the spam threshold.
💡
Key Insight Logistic regression minimises binary cross-entropy: $-[y\log\hat{p} + (1-y)\log(1-\hat{p})]$. This is identical to the loss used in every neural network classifier output layer. Logistic regression IS a one-layer neural network with sigmoid activation — just without hidden layers.
04Intermediate

Decision Trees & Random Forests

Decision trees make predictions by asking a series of yes/no questions about features. Random Forests train hundreds of trees on random data subsets and average their votes — turning a notoriously fragile model into one of the most robust algorithms in all of ML.

Decision tree — loan approval classifier
python 
from sklearn.tree import DecisionTreeClassifier
from sklearn.ensemble import RandomForestClassifier

# Single tree — interpretable but overfits without constraints
tree = DecisionTreeClassifier(max_depth=5, min_samples_leaf=10)

# Random Forest — robust, great out-of-the-box default
rf = RandomForestClassifier(
    n_estimators=300,      # number of trees
    max_features="sqrt",  # random feature subset per split
    n_jobs=-1             # use all CPU cores
)
rf.fit(X_train, y_train)

# Feature importances — which features matter most?
for name, imp in sorted(zip(feature_names, rf.feature_importances_),
                        key=lambda x: -x[1])[:5]:
    print(f"  {name}: {imp:.3f}")

✓ Use when

  • Tabular / structured data
  • Mix of numeric and categorical features
  • Need feature importance rankings
  • Want a reliable baseline that "just works"

✗ Don't use when

  • Images, text, audio (use DL)
  • Very fast inference required (try XGBoost/LightGBM)
  • Full interpretability needed (single tree is better)
🏢 Real-world: Airbnb's dynamic pricing uses gradient-boosted trees (XGBoost — RF's cousin) on 100+ tabular features: location, season, review score, nearby events, and booking lead time.
💡
Key Insight Random Forests beat single trees via the bias-variance tradeoff. Each deep tree overfits (high variance). Averaging 300 trees that each see different random data/features cancels the variance, leaving only signal. This technique is called bagging (Bootstrap AGGregating).
05Beginner

k-NN and k-Means

Two elegantly simple algorithms sharing one core idea: distance between points carries meaning. k-NN uses neighbours to classify new points (supervised). k-Means finds natural groups by iteratively moving cluster centres (unsupervised). Together they teach the concept of similarity in feature space.

k-NN (k=3) — classify the triangle ▲
k-Means (k=3) — find cluster centres ⊙
python 
from sklearn.neighbors import KNeighborsClassifier
from sklearn.cluster import KMeans

# k-NN: supervised — memorises all training data
knn = KNeighborsClassifier(n_neighbors=5, metric="euclidean")
knn.fit(X_train, y_train)
pred = knn.predict(X_test)   # O(n) lookup at inference!

# k-Means: unsupervised — assign each point to nearest centroid
km = KMeans(n_clusters=3, n_init=10, random_state=42)
km.fit(X)
labels    = km.labels_           # cluster id per point
centroids = km.cluster_centers_  # shape (k, n_features)
# Use elbow plot on km.inertia_ to choose k

✓ Use when

  • k-NN: small datasets, similarity search, anomaly detection
  • k-Means: customer segmentation, image colour quantisation
  • Great for prototyping — minimal hyperparameter tuning

✗ Don't use when

  • k-NN: large datasets (O(n) per query — use approximate NN)
  • k-Means: non-spherical clusters or unknown k
  • High-dimensional raw features (use UMAP/PCA first)
🏢 Real-world: Spotify uses k-Means to cluster songs by audio features (tempo, energy, danceability) into genre groups, then recommends within the same cluster as songs you've liked.
💡
Key Insight The "curse of dimensionality" kills both algorithms in high dimensions — all points become equidistant, so "nearest neighbour" loses meaning. Always reduce dimensions first (PCA, UMAP, or learned embeddings) before using distance-based methods on more than ~50 features.
06Intermediate

Overfitting & Underfitting

The most common reason ML projects fail in production. Overfitting means the model memorised the training data and can't generalise. Underfitting means the model is too simple to capture the true pattern. Finding the sweet spot is the central challenge of supervised learning.

Underfit (High Bias)

Too simple — flat line on a curve. High train AND test error.

Good Fit ✓

Captures the true signal. Low train error, low test error. Generalises.

Overfit (High Variance)

Memorised noise. ~0 train error, high test error. Fails in production.

Bias-variance tradeoff — the classic learning curve
python 
from sklearn.linear_model import Ridge, Lasso

# L2 (Ridge): penalises large weights — shrinks all towards zero
ridge = Ridge(alpha=1.0)   # higher alpha = stronger penalty

# L1 (Lasso): drives some weights exactly to zero
# Use for automatic feature selection
lasso = Lasso(alpha=0.1)

# For trees: constrain depth to limit complexity
# DecisionTreeClassifier(max_depth=5, min_samples_leaf=10)

# Neural nets: dropout randomly zeros neurons during training
# import torch.nn as nn; nn.Dropout(p=0.3)

# Early stopping: stop when val loss stops improving
# model.fit(X_train, y_train, early_stopping_rounds=10)

Signs of overfitting

  • Train accuracy 98%, val accuracy 72%
  • Val loss rises while train loss still falls
  • Wildly different scores across CV folds

Fixes for overfitting

  • More training data or data augmentation
  • Regularisation: L1, L2, dropout, early stopping
  • Simpler model: less depth, fewer parameters
🏢 Real-world: A fraud detection model showed 99.9% training accuracy but 60% production accuracy — it overfit to specific card numbers in training data. Fix: remove card-specific IDs, add L2 regularisation, retrain.
💡
Key Insight Total error = bias² + variance + irreducible noise. You can't eliminate all three. Regularisation and more data shift the balance — but the irreducible noise floor is a hard limit set by how noisy the underlying process is. No model can beat it.
07Intermediate

Cross Validation

A single train/test split gives you one estimate of performance — which can be lucky or unlucky. Cross-validation gives a statistically robust estimate by rotating through every part of the data as the test set, so every example gets evaluated exactly once.

5-fold cross validation — each fold takes a turn as the test set
Mean CV score: 0.909 ± 0.014 std — low std = stable model ✓
python 
from sklearn.model_selection import cross_val_score, StratifiedKFold
from sklearn.ensemble import RandomForestClassifier
import numpy as np

model = RandomForestClassifier(n_estimators=200)

# StratifiedKFold preserves class ratios in each fold
cv     = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(model, X, y, cv=cv, scoring="roc_auc")

print(f"AUC: {scores.mean():.3f} ± {scores.std():.3f}")
# High std → model is sensitive to which data it sees
# → add more data or use a simpler / more regularised model
🏢 Real-world: Kaggle competitors always use 5-fold CV to compare models without ever touching the test set. A model scoring 0.91 ± 0.01 on CV is far more trustworthy than one scoring 0.93 on a single lucky split.
💡
Key Insight The standard deviation across folds is as important as the mean. 0.91 ± 0.01 beats 0.93 ± 0.08 in production. High std means your model's performance depends on which specific rows it trains on — a sign of instability that will show up as unreliable live metrics.

Accuracy is almost always the wrong metric. A model that predicts "not fraud" for every transaction achieves 99.9% accuracy on most fraud datasets — and catches zero fraud. The right metric depends on the specific cost of false positives vs false negatives in your domain.

Which metric to use when?
Fraud Detection
Use Recall — missing fraud (FN) is far worse than a false alarm
Spam Filter
Use Precision — good email in spam (FP) destroys trust
Imbalanced Classes

python 
from sklearn.metrics import (
    classification_report, roc_auc_score,
    confusion_matrix, ConfusionMatrixDisplay
)

y_pred  = model.predict(X_test)
y_proba = model.predict_proba(X_test)[:, 1]

# One call gives precision, recall, F1 per class
print(classification_report(y_test, y_pred))

# AUC uses probabilities (more informative than accuracy)
auc = roc_auc_score(y_test, y_proba)

# Adjust threshold to trade precision vs recall
threshold  = 0.3           # lower → catch more positives (↑ recall, ↓ precision)
y_adjusted = (y_proba > threshold).astype(int)

Precision vs Recall

  • Precision: "Of all I flagged positive, how many were right?"
  • Recall: "Of all actual positives, how many did I catch?"
  • They trade off — lowering threshold raises recall, lowers precision

When NOT to use accuracy

  • Class imbalance > 10:1 (use F1 or PR-AUC)
  • Different costs for FP vs FN (tune threshold)
  • Ranking problems (use AUC or NDCG)
🏢 Real-world: A cancer screening model at 97% accuracy sounds impressive — until you learn 97% of patients are healthy so predicting "healthy" for everyone hits 97%. ROC-AUC of 0.72 revealed the truth.
💡
Key Insight ROC-AUC answers one question: if I randomly pick one positive and one negative example, what's the probability my model ranks the positive higher? AUC of 1.0 = always correct. AUC of 0.5 = coin flip. It's threshold-independent, so it measures the model's intrinsic discriminative ability regardless of the decision cutoff.

MAE

Mean Absolute Error — average absolute difference. Robust to outliers. Interpretable in original units.

RMSE

Root Mean Squared Error — penalises large errors more heavily. Use when big mistakes are especially costly.

R² Score

Fraction of variance explained. R²=1 is perfect. R²=0 means model ≈ mean baseline. R²<0 means worse than baseline.

Macro vs Weighted F1

Macro: equal weight per class — use for imbalanced data. Weighted: weight by class size — use for balanced data.

PR-AUC

Precision-Recall AUC. Preferred over ROC-AUC for highly imbalanced datasets — focuses only on minority class performance.

Log Loss

Penalises confident wrong predictions heavily. Use when you need well-calibrated probabilities, not just rankings. Lower is better.
