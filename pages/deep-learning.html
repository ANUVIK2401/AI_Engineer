01Beginner

Neural Network Basics — Forward Pass

A neural network is a composition of linear transformations and non-linear activations. Data flows in one direction during inference — the forward pass. Each layer transforms the representation until the output layer produces a prediction.

Animated forward pass — signals propagate left → right through layers
Forward pass — single layer
$\mathbf{h}^{(l)} = \sigma\!\left(\mathbf{W}^{(l)}\mathbf{h}^{(l-1)} + \mathbf{b}^{(l)}\right)$

Weights W

Learned parameters. A matrix that linearly mixes the previous layer's activations. Shape: [out_features × in_features].

Bias b

Shifts the activation threshold, letting the network learn a non-zero intercept. Without it, every hyperplane passes through the origin.

🔧
Why an AI Engineer needs this When you trace a model (model.forward(x)), profile latency per layer, or implement a custom module in PyTorch, you are executing this formula. Shape errors in production models (RuntimeError: mat1 and mat2 shapes cannot be multiplied) all trace back to mismatched W dimensions.
02Beginner

Activation Functions

Without non-linearity a stack of linear layers collapses to one linear transformation — no matter how deep. Activations inject non-linearity after each layer so the network can learn curved decision boundaries. The choice matters for gradient flow and training stability.

Interactive — click a function to see its curve
ReLU — f(x) = max(0, x). Default for hidden layers. Fast, sparse activations, but prone to dying neurons when weights go permanently negative.
FunctionFormulaOutput rangeUsed inKey weakness
ReLUmax(0, x)[0, ∞)CNNs, MLPsDying neurons
Sigmoid1 / (1+e⁻ˣ)(0, 1)Binary output, LSTM gatesVanishing gradient
Tanh(eˣ−e⁻ˣ) / (eˣ+e⁻ˣ)(−1, 1)RNNs, LSTMsVanishing gradient
GELUx · Φ(x)(−0.17, ∞)BERT, GPT, LLaMA, all LLMsSlightly slower
🔧
Why an AI Engineer needs this When you open a Hugging Face model config and see "hidden_act": "silu" or "gelu", you understand why that matters. GELU is the dominant activation in modern LLMs because it outperforms ReLU on language tasks; LLaMA uses SiLU (a close relative). Accidentally replacing it in a custom model build degrades quality silently.
03Intermediate

Backpropagation & the Chain Rule

Backprop is the algorithm that makes training deep networks tractable. It computes the gradient of the loss with respect to every weight by applying the chain rule backwards through the computational graph — then an optimiser (SGD, Adam) nudges each weight in the direction that reduces the loss.

Step-by-step walkthrough
1
Forward Pass — compute predictions and cache activations
Run input through every layer to get ŷ. Store intermediate values h⁽ˡ⁾ — you will need them during the backward pass.
2
Compute Loss L
Measure how wrong the prediction is. Cross-entropy for classification, MSE for regression. This is a scalar you differentiate.
3
Backward Pass — chain rule through every layer
Compute ∂L/∂W for every weight by multiplying local gradients backwards layer by layer. PyTorch autograd does this automatically via a dynamic computational graph.
4
Update Weights
Optimiser applies: W ← W − η · ∂L/∂W. Adam also tracks first and second moment estimates for adaptive learning rates per parameter.
Chain rule — gradient through composition
$$\frac{\partial L}{\partial W^{(l)}} = \frac{\partial L}{\partial h^{(l)}} \cdot \frac{\partial h^{(l)}}{\partial z^{(l)}} \cdot \frac{\partial z^{(l)}}{\partial W^{(l)}}$$
💡
Vanishing & Exploding Gradients Multiply many small numbers (gradients < 1) → product → 0 (vanishing). Multiply large numbers → ∞ (exploding). Residual connections let gradients bypass layers. Layer norm keeps activations in a stable range. Gradient clipping caps magnitude. LLMs use all three.
🔧
Why an AI Engineer needs this When fine-tuning fails — loss doesn't decrease, NaN loss, weights diverge — the root cause is almost always a gradient problem. Understanding backprop tells you why gradient checkpointing, mixed-precision training, and learning rate warmup exist, and when each is required.
04Intermediate

Convolutional Neural Networks (CNNs)

CNNs exploit spatial locality — a small kernel slides across the input, sharing weights at every position. This gives translation invariance and massive parameter savings vs a fully-connected approach processing raw pixels. Foundational for image, audio, and patched-input models.

Stride

Pixels the kernel jumps per step. Stride 2 halves spatial dimensions — equivalent to pooling without extra parameters.

Padding

Zeros added around the border. "Same" padding preserves input size; "valid" lets it shrink by (kernel−1) per side.

Channels

Each filter produces one output channel. 64 filters → 64 feature maps. Depth grows as spatial dims shrink across layers.

🔧
Why an AI Engineer needs this Vision encoders in multimodal LLMs (CLIP, LLaVA, GPT-4V, Gemini) use CNN stems or patch-convolution layers. Choosing image resolution, patch size, or vision backbone in a multimodal pipeline means directly setting convolution hyperparameters. You also need this to understand why ViT patchification is a single convolution.

Before Transformers, recurrent networks were the backbone of all sequence modelling. An RNN maintains a hidden state h that carries information across timesteps. 💡
Why Transformers replaced RNNs RNNs process tokens sequentially and cannot parallelise across the sequence. A 4096-token input needs 4096 sequential steps. On GPUs built for parallel computation, this is catastrophic for throughput. Transformers process all tokens simultaneously — 100× faster training at the scales required for LLMs.
🔧
Why an AI Engineer needs this LSTM-style gating reappears in Mamba (state-space models) and linear attention variants. If you are evaluating whether to use an SSM over a Transformer for long-sequence inference tasks — where SSMs are O(1) per step vs O(n) KV cache growth — you need to understand the LSTM heritage and what SSMs trade off to achieve it.
06 Advanced ⭐ Foundation

Transformer Architecture — The Foundation of Everything

Introduced in "Attention Is All You Need" (Vaswani et al., 2017), the Transformer replaced recurrence entirely with self-attention. Every modern LLM (GPT, LLaMA, Gemini, Claude), vision model (ViT), and multimodal system is a Transformer variant. This is the single most important architecture an AI engineer must understand.

Encoder-only (BERT, RoBERTa): bidirectional context, good for classification and embeddings. Decoder-only (GPT, LLaMA, Claude): autoregressive generation, causal masking. Encoder–Decoder (T5, BART): seq2seq tasks like translation, summarisation.

Attention Is All You Need — the core formula (Vaswani et al., 2017)
$$\text{Attention}(Q,\, K,\, V) \;=\; \text{softmax}\!\left(\frac{QK^{\top}}{\sqrt{d_k}}\right)V$$

Q — Query

"What am I looking for?" — the current token's representation of its information need. Projects the input with W^Q.

K — Key

"What do I contain?" — every token advertises its content. Q·Kᵀ scores how well each query matches each key.

V — Value

"What do I contribute?" — the actual information vector retrieved, weighted by the softmax-normalised attention scores.

💡
Why √d_k scaling? For large d_k (head dimension), Q·Kᵀ dot products grow proportionally in magnitude, pushing softmax outputs toward 0 or 1 — regions with near-zero gradients. Dividing by √d_k keeps variance ≈ 1 regardless of head dimension, preventing gradient saturation and enabling stable training at depth.
Multi-Head Attention

Instead of a single attention function, run h attention heads in parallel, each with independent W_Q, W_K, W_V projection matrices. Each head attends to different positional offsets and semantic relationships simultaneously. Outputs are concatenated and projected back to model dimension d_model.

$$\text{MultiHead}(Q,K,V) = \text{Concat}(\text{head}_1,\,\ldots,\,\text{head}_h)\,W^O$$ $$\text{head}_i = \text{Attention}\!\left(QW_i^Q,\; KW_i^K,\; VW_i^V\right)$$

For LLaMA-3-8B: d_model=4096, h=32 heads, d_k=128 per head. Total Q/K/V parameters ≈ 50M per layer.

Positional Encoding — how order is injected

Self-attention is permutation-invariant — it treats tokens as a set, not a sequence. Positional encodings inject order by adding position-dependent signals to token embeddings before attention.

Feed-Forward Network (FFN)

After attention (cross-token mixing), each token passes through an independent 2-layer MLP. No cross-token interaction — the FFN expands to 4×d_model then contracts. This is where most factual "knowledge" is thought to be stored in LLMs (rows of W2 act as memory slots).

$\text{FFN}(x) = \text{GELU}(xW_1 + b_1)\,W_2 + b_2$

Modern LLMs use SwiGLU (LLaMA, Gemma, Mistral): FFN(x) = (xW₁ ⊗ SiLU(xW₃))W₂ — an extra gating projection that improves perplexity at matched parameter count.

Residual Connections & LayerNorm

Each sublayer wraps with a skip connection: output = x + Sublayer(LayerNorm(x)) (Pre-Norm, preferred by modern LLMs). Residuals let gradients flow unchanged through depth, solving vanishing gradients at 32–96 layers. Pre-Norm is more training-stable than the original Post-Norm formulation.

Causal Masking (Decoder)

Autoregressive generation requires that position t cannot attend to positions > t. This is implemented by adding −∞ to attention logits for future positions before softmax — the mask is a lower-triangular matrix. During inference, only one new token is generated per forward pass using the KV cache.

🔧
Why an AI Engineer needs this Every consequential LLM decision flows from here: choosing a model (encoder vs decoder), setting context length (tied to positional encoding), enabling Flash Attention, configuring KV cache, applying paged attention for serving, selecting quantisation granularity per layer, implementing LoRA on attention projections. You cannot make sound engineering decisions about LLMs without understanding the Transformer block thoroughly.

08Intermediate

Transfer Learning — Pretrained vs Fine-tuned

Training a Transformer from scratch costs millions of dollars and weeks of GPU time. Transfer learning reuses a pretrained model's weights as a starting point, fine-tuning only what is needed for the new task. This is the core workflow behind every LLM fine-tune, LoRA adaptation, and RLHF pipeline.

💡
LoRA — parameter reduction via low-rank decomposition Instead of updating W directly (d×d parameters), LoRA freezes W and learns ΔW = BA where B ∈ ℝ^(d×r) and A ∈ ℝ^(r×d), with rank r ≪ d. For r=8 on a 4096×4096 weight matrix: parameters drop from 16.7M → 65k — a 257× reduction. At inference the adapter merges back: W' = W + BA, adding zero latency.
🔧
Why an AI Engineer needs this When a customer asks "can we make the model behave differently for our domain?", transfer learning is your first answer. Knowing the full spectrum from prompt engineering to full fine-tune lets you right-size the solution: LoRA on a 7B base often beats a full fine-tune on a smaller model, costs a fraction of the compute, and produces a mergeable adapter that adds zero serving latency.
